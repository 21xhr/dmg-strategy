# Analysis Folder Guide

This folder stores analytical artifacts, generated exports, and valuation material that support strategy, fundraising, and external review.

## Recommended structure

- `context-exports/`
  Stores generated GPT-ready exports and pass files.
- `valuation-reports/`
  Stores valuation memos and analytical reports.

## What belongs here

- GPT export dumps
- staged pass files for analysis
- valuation drafts and final reports
- one-off analytical material that helps thinking or fundraising

## What does not belong here

- core product documentation that describes the product itself
- day-to-day roadmap items that belong in strategy docs
- reusable fundraising materials such as one-pagers or advisor briefs

Those belong in their own dedicated directories.

## Naming principle

Use explicit names that answer three questions immediately:

1. what is this file for?
2. who is it for?
3. when was it produced?

Examples:

- `gpt-pass-a-context-2026-04-05.txt`
- `valuation-vc-positioning-2026-04-05.md`
- `demo-baseline-manifest-2026-04-06.md`
