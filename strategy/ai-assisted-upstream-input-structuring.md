# AI-Assisted Upstream Input Structuring

## Purpose

This note defines an important product-positioning boundary.

AI-assisted input structuring can be a real DMG capability and a meaningful selling point.

It should not be confused with the whole product promise.

## Core rule

AI can strengthen DMG when it helps turn messy raw expression into more reliable, more comparable, and more decision-ready upstream inputs.

That keeps AI inside the input-structuring layer.

It does not require DMG to become a generic AI note-taking or recommendation product.

Delegated judgment can still exist later as a constrained downstream feature in some flows.

It should not be the main promise, the default framing, or the first sales layer.

## Why this matters

Many organizations already classify information into named document families, summaries, reporting channels, and operational tools.

That does not automatically mean they already have strongly structured, comparable, decision-ready upstream data.

Named document types and existing synthesis habits are not the same thing as disciplined input shape.

The stronger claim is narrower:

- the organization may already know where information is supposed to go
- the content may still arrive in inconsistent forms
- repeated issues may still be hard to compare across sites, roles, or time
- staff may still spend time manually reformatting material before it becomes decision-usable

## Product implication

DMG stays aligned with its purpose when AI is used to help produce the expected upstream shape.

That can include:

- converting a meeting or field exchange into a structured draft
- preserving role-distinct inputs instead of flattening them too early
- drafting into the expected fields for a given flow or document family
- making repeated issues easier to compare across dispositifs, teams, or cases
- reducing manual recopy between field discussion and decision-facing material

This is still upstream input structuring.

It is not merely "better summaries".

## Configuration and operator value

The selling point is stronger when the tenant or operator helps define the expected shape of the received information.

In that model, DMG does not just capture text and summarize it.

It helps co-define and enforce input structure per flow.

Examples:

- one GEPP-like flow may need issue category, role, recurrence, dispositif, and escalation need
- one customer-demand flow may need requester type, willingness to support, urgency, and execution dependency
- one incident or risk flow may need severity, location, operational impact, and required arbitration level

That matters commercially because it shows the operator that integration work produces a tenant-specific governance asset rather than a generic AI add-on.

## AI provider boundary

DMG can still benefit from external AI providers inside this model.

The important rule is that external AI should strengthen DMG's ability to structure inputs upstream rather than redefine the product as a thin wrapper around generic AI summarization.

Good use cases include:

- speech-to-structured-draft capture
- structured synthesis into expected fields
- extraction help for recurring issue patterns
- formatting acceleration for decision-facing inputs

Weak use cases include:

- generic meeting summary with no downstream structure because readable output alone does not improve the shape of decision-facing inputs
- recommendation-first positioning before input discipline exists because more AI on top of weak flows usually produces more noise rather than more decision-grade signal
- rhetorical drift toward delegated judgment as the main promise even if constrained delegated-judgment features may later become useful in narrow contexts

## Audience-ready versions

### Investor-facing paragraph

DMG can use external AI capability where useful, but the defensible layer is not generic AI summarization. The stronger asset is the tenant-specific structure around how raw field signal becomes reliable, comparable, decision-ready input inside real workflows. That makes AI-assisted upstream input structuring a meaningful accelerator without reducing the company to a thin wrapper around commodity models.

### Operator-facing paragraph

DMG does not need to replace existing tools or invent a new document culture from scratch. It can help define and enforce the decision-usable shape of the information that already moves through existing flows, so what comes out of meetings, field exchanges, and recurring issues is easier to compare, escalate, and arbitrate without extra manual reformatting.

## Concrete flow example

### GEPP-to-codir structured draft

One AAJD-shaped example is a GEPP discussion that surfaces a recurring issue at dispositif level and needs to become decision-usable at codir level.

Raw source material may include role-distinct inputs from a chauffeur, maitresse de maison, secretaire, educateur, psychologue, and manager.

A useful DMG-assisted draft would not flatten that into one generic paragraph.

It would preserve and structure the signal into fields such as:

- issue label
- dispositif or site
- roles reporting the issue
- recurrence window
- concrete observed effects
- what has already been tried locally
- whether the same pattern appears in other dispositifs
- operational risk if the issue stays local
- budget, HR, systems, or governance dependency
- requested arbitration level

That kind of draft is stronger than a generic summary because codir can compare cases across sites, see whether the issue is isolated or repeated, and decide whether the right response is local adjustment, cross-service coordination, budget review, systems change, or formal escalation.

## Investor-facing implication

This can be a meaningful investor-facing point when presented correctly.

The attractive claim is not simply that DMG "has AI".

The stronger claim is that DMG can absorb external AI capability where useful while keeping product value in:

- tenant-specific input structure
- workflow fit
- decision-boundary clarity
- reusable governance configuration

That is stronger than a commodity AI-wrapper story because the defensible layer is not the raw model alone.

It is the structured governance surface built around the tenant's real decision flows.

## Review rule

Reject product framing that:

- treats AI summarization as the whole DMG promise
- confuses named document families with already-structured decision-grade inputs
- implies that better synthesis automatically means delegated decision authority
- hides the fact that the main asset is the operator-defined shape of decision-usable input
