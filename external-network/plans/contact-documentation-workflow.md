# Contact Documentation Workflow

## Purpose

This note defines the default documentation updates that should happen when work concerns external network contacts, outreach, discovery calls, or post-call synthesis.

It exists so contact work updates the durable note, the operational tracker, and the broader thesis only when each layer actually changed.

The tracked people include market-learning contacts, strategic reviewers, operator interviews, warm introductions, and pilot-adjacent relationships.

## Core rule

When a request concerns a specific tracked contact, do not stop at ad hoc advice.

Convert the request into durable contact documentation and update the operational tracker if the conversation changed the next move.

## Default update set

### 1. Contact note

Create or update a note under `external-network/network-contacts/<Contact>/` when the request adds durable information such as:

- relationship context that is easy to lose
- discovery-call notes
- language corrections from the contact
- decision-process details
- confirmed follow-up paths

Keep this note richer than the tracker.

Use it to preserve what should still matter several conversations later.

### 2. Current network action sheet

Update `external-network/plans/current-network-action-sheet.md` when any of these changed:

- status
- next contact step
- primary motion
- signal to extract
- secondary gains that now matter operationally

Keep this row action-oriented.

Do not turn it into a meeting transcript.

### 3. Current network context table

Update `external-network/network-contacts/current-network.md` only when the relationship meaning or the unusually valuable context changed.

Do not copy operational next steps there.

### 4. Market-feedback and LOI plan

Update `fundraising/plans/market-feedback-and-loi-plan.md` only when the learning changes a cross-contact thesis, cluster framing, or the definition of good signal.

Do not update it for every routine meeting note.

## Language rule

Keep each document linguistically consistent.

Do not mix English and French inside the same note unless preserving a direct quote, an official proper name, or a term that is immediately defined.

Use `external-network/network-contacts/current-network.md` as the source of truth for prospect language, and keep the durable note, follow-up draft, trame, and summary sections in that language.

Preserve contact wording when it sharpens the thesis.

If a contact rejects a term, record the replacement language and prefer it in downstream follow-up.

## Evidence rule

Do not overclaim pain.

If the contact says the decision process works reasonably well, preserve that constraint and narrow the wedge accordingly.

## Follow-up rule

When a contact opens a concrete next path, capture the named people, systems, or artifacts to inspect next.

These follow-ups often carry stronger signal than another generic pitch conversation.
